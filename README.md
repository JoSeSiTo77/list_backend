# Commit convention

## Types

1. feat: #ID, new functionality.
    - Description commit

* Important Change: Indicates that the commit breaks backward compatibility.
2. feat!: #ID, new functionality.
    - Description commit
    * What's change:
    [OPTIONAL]: BREAKING CHANGE: <description> 

3. fix: #ID, fixing errors.
    - Description commit

4. refactor: #ID, internal improvement without changing behavior.
    - Description commit

5. docs: #ID, documentation.
    - Description commit
